Steps within the AccountingTokenizer Class¶

  1. Accounting and auditing words are loaded into the word matcher for detecting them in texts. They are not stematized in this step so that their uniqueness is maintained. For example, 'accounts receivable'. If this word is stematized first, it will cause a problem when we try to identify a two-word phrase, because we will have account and receiv separately. Thus, the dictionay will not be able to match it.
  2. Accounting and auditing words with s,es,ed are loaded into the word matcher.
  3. Texts are tokenized.
  4. The tokenized texts are then matched with the word list stored in the word matcher to find all the accounting words and their locations in the texts.
  5. The tokenized texts are stemmatized.
  6. The matched accounting and auditing words are stemmatized.
  7. The stemmatized tokens of the text are replaced with accounting and auditing tokens corresponding to their detected locations. Partial tokens from multi-word phrases are then dropped.

Step within the replaceMatch Function¶

  1. Next, I deal with synonyms and acronyms with s,es,and ed by stemmatized them and match them back to the outcome from step 7.

These steps produced the final output of stmmatized tokens of texts.

def clean_text_traditional(text): 
    """ Converting all to lower case, splitting words, dropping punctuations, grouping,
    tokenizing (unigram) and porter stemming text
    """
    text = "".join([word.lower() for word in text if word not in string.punctuation])
    text = re.split('\W+', text)
    text = [ps.stem(word) for word in text if word not in stopwords]
    
    return text

# Spacy Model: Create tools to deal with auditing and accounting words
# ========================================================================

# Load a spacy model
nlp        = spacy.load('en_core_web_sm')

# Import our wordlist
wordlist   = pd.read_excel("..\\input\\combined_wordlist.xlsx",sheet_name="Sheet1")
# Select only the original word column
# accounting = list(wordlist.Word) 
accounting = list(wordlist.Word)+list(map(lambda x:x+'s',wordlist.Word))+list(map(lambda x:x+'es',wordlist.Word))+list(map(lambda x:x+'ed',wordlist.Word))

# Create a list of stopwords + punctuation + special characters in texts
unwanted   = stopwords + list(string.punctuation)+['\n','','s','','','','','\n ','\n  ','\n\n',"'s",'']


class AccountingTokenizer:

    def __init__(self,nlp,accounting):
        self.nlp        = nlp
        
        # Prepare a matcher for adding accounting and auditing words
        self.matcher    = PhraseMatcher(nlp.vocab)
        self.accounting = accounting
        self.patterns   = list(nlp.pipe(self.accounting))
        
        # Add accounting and auditing words
        self.matcher.add('ACC', None, *self.patterns)
        self.accMatch   = []

    def tokenizeIt(self,text):
        doc             = nlp(text.lower().lstrip())          # Remove leading space
        token           = [ps.stem(t.text) for t in doc]      # Tokenizing
        matches         = self.matcher(doc)                   # Apply our matcher to text 
        
        # Check which accounting or auditing words are matched and keep only stems
        self.accMatch   = [ps.stem(str(doc[start:end])) for match_id, start, end in matches] 
        
        # Create indices for accounting or auditing words for cleaning purpose
        m_ind           = [(start,end) for match_id, start, end in matches]        
        ll = []
        
        # Remove unused partial words and replace them with full stem words
        for i in m_ind:
            ll = ll + list(range(i[0]+1,i[1]))
        for i in range(len(self.accMatch)):
            token[m_ind[i][0]] = self.accMatch[i]
        matched         = [i for j, i in enumerate(token) if j not in ll]  
        
        return [w for w in matched if w not in unwanted]

    
def replaceMatch(wordlist,raw_tokens):
    
    # Main Wordlist: for speeding purpose, go through the rows with either an acronym or synonym only
    wordlist2 = wordlist[pd.notnull(wordlist.Acronym) |pd.notnull(wordlist.Synonym)].copy() 
    
    # Prepare the dictionary for matching  
    keyword_dict = {}
    for i in wordlist2.index:
        ''' Start mapping the stems of acronyms and synonyms to the stems of their original words
            By doing this, my argument is, it will take care of s, es, and ed forms of the acronyms 
            synonyms at the same time. In effect, we don't need to increase workloads to the computer from
            expanding our mapping list to cover words with s,es, and ed, since the stems of those words with 
            s,es, and ed should be the same as the stems of the acronyms and synonyms.'''
        
        keyword_dict.update({ps.stem(wordlist.loc[i,'Word']):[ps.stem(str(wordlist.loc[i,'Acronym'])),
                                                              ps.stem(str(wordlist.loc[i,'Synonym']))]})
    # Clean up the dictionary for mapping by removing nan
    keyword_dict = {k:[elem for elem in v if elem is not np.nan] for k,v in keyword_dict.items()} 

    # Build our Keyword Processor by adding the word list of acronyms and synonyms above into it
    myKeys = KeywordProcessor()
    myKeys.add_keywords_from_dict(keyword_dict)
    
    
    return [myKeys.replace_keywords(t) for t in raw_tokens]

My functions do the following things:

1) Matching synonyms and acronyms back to their original words. For example, abc method is matched to activity based costing method.

2) Take care of multiple-word phrases. Ex. when we tokenize accounts receivable, we want to have 'accounts receiv' not 'account' and 'receiv' separately.

3) Tokenization

4) Extract stems of all tokens. Ex accounts receivable => account receiv
